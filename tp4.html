Git

  • Dépôt distant (sur gitlab ou github) - partagé entre les utilisateurs
  • Dépôts locaux - sur chacune de vos machines

Commandes git

  • git add .
    Ajoute tous les fichiers et dossiers du répertoire à l'index git
  • git commit -m "message"
    Ajoute un commit local
  • git remote add origin "MON_URL"
    Envoie le code de votre dépôt local vers le dépôt distant configuré
  • git push -u origin master
    Envoie le code de votre dépôt local vers le dépôt distant configuré
  • git pull
    Récupérer le code du dépôt distant sur votre dépôt local = git fetch + git merge
  • git clone url_gitlab nom_dossier
    Clone le dépôt distant dans un dépôt local sur votre machine dans le dossier "nom_dossier"

Quelques conseils pour git

  • Faites des commits lorsque vous avez avancé sur chaque fonctionnalité
  • Évitez de travailler tous sur les mêmes classes, les mêmes fichiers, sinon vous aurez beaucoup de conflits : répartissez / séparez le travail

Tests unitaires

Teste une méthode, un comportement précis de l'application, généralement plusieurs parties :
  • Initialisation, "mise en situation" : on génère les données utiles pour le test (BDD ou objets python)
  • Traitement : on applique la méthode, le traitement à tester
  • Assertion : on vérifie que le traitement donne le résultat voulu
  • Nettoyage post-test : suppression des objets créés en BDD ou autres

Tests

  • Les tests unitaires garantissent qu'une méthode fonctionne dans un cas très précis
  • Multiplier les tests pour inclure plus de comportements
  • Prendre en compte les autres types de tests
